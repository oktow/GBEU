      belum ada di sprite frames)

    - unlock_cost = 500, is_unlocked = false

File Dimodifikasi:
  script/player.gd
    - Baris 45-47: tambah @export_group("Combat Style") + vars:
        current_attack_style: AttackStyle
        player_exp: int
    - Baris 59: hapus combo_damage dict (digantikan resource)

        combo_end_frames dari current_attack_style

        & animation_name dari resource (dinamis, bukan hardcode "attack1")

Sesi 5: Modular Enemy System dengan Resource-Driven Configuration
===============================================================================

Latar Belakang:
- Enemy (ghost & dusar) masih pakai stat hardcode: health, damage, speed
  langsung di script.
- Perlu sistem resource-driven agar tuning game balance bisa dilakukan
  via Inspector tanpa edit kode.
- Dusar perlu AI ranged (kiting, projectile burst, teleport) yang
  konfigurasinya fleksibel.
- Player juga butuh PlayerConfig terpusat menggantikan hardcode values
  di player.gd.
- Enemy butuh floating health bar yang muncul saat kena hit.

File Baru:
  script/core_stats.gd

    - Field: enemy_name, health, damage, speed, radar_radius,
             attack_cooldown, knockback_force, stun_duration,
             kill_score_value, special_energy_value

    - Field: patrol_range, patrol_speed_mult, stop_distance, jump_force,
             double_tap_delay

  script/projectile_data.gd

    - @export core_stats, projectile_data, teleport_data: Resource

    - Inspector: player_config = gendra_config.tres
    - Animasi sprite baru: jump_kick, charging_special, walk_carry,

Arsitektur:
  Semua data game dipisah dari script ke Godot Resources:
    CoreStats     → stat dasar enemy
    PatrolData    → movement patrol melee enemy
    ProjectileData  → ranged attack config
    TeleportData  → teleport behavior config
    PlayerConfig  → semua tuning parameter player

  Pattern yang dipakai:
    - @export_group() untuk Inspector rapi
    - @export_range(min, max, step) + tooltip Bahasa Indonesia (px/dtk, dtk)
    - Resource file di assets/data/ dengan prefix snake_case
    - Enemy AI: state machine (enum untuk Dusar, boolean flag untuk Ghost)
    - HUD auto-hide dengan display_duration configurable
    - Radar cooldown cegah target flickering
    - Camera-bounded movement untuk kiting & teleport
    - die() flow terstandarisasi: register kill → disable → reward → drops → anim → free

  script/player.gd
    - Baris 52-64: _ready() detek PlayerState.should_restore →
      restore state + clamp ke max config, atau init default
    - Baris 510: die() panggil PlayerState.reset()

  script/stage_001.gd

  script/main_menu.gd
    - Baris 15: _on_play_pressed() panggil PlayerState.reset()
    - Baris 25: _on_survival_pressed() panggil PlayerState.reset()

    - Baris 266: _on_exit_pressed() panggil PlayerState.reset()

    - Baris 30: _ready() panggil PlayerState.reset()

Alur Persistence:
  Stage 1 → player ambil damage, stamina turun, beli Brutal Attack →
  trigger NextStageTrigger → PlayerState.save(body) → change_scene →
  stage_002 _ready() → PlayerState.should_restore == true →
  restore health, stamina, special, attack_style → clamp ke max config →
  player lanjut dengan state Stage 1.
