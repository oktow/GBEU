 ASSERT METHODS YANG SERING DIPAKAI
================================================================================

  assert_eq(actual, expected, msg)       -- equal
  assert_ne(actual, not_expected, msg)   -- not equal
  assert_true(actual, msg)               -- true
  assert_false(actual, msg)              -- false
  assert_has(dict_or_arr, key, msg)      -- key exists
  assert_does_not_have(...)              -- key not exists
  assert_null(actual, msg)
  assert_not_null(actual, msg)
  assert_between(val, low, high, msg)

================================================================================
 CARA MENJALANKAN TEST
================================================================================

--- CARA 1: Via Godot Editor (termudah) ---
  1. Buka project di Godot
  2. Menu: Project > Tools > GUT > Configure
  3. Isi "Test Directories" dengan:  res://test
  4. Klik Close
  5. Menu: Project > Tools > GUT > Run
  6. Panel GUT akan muncul dengan hasil test (hijau = pass, merah = fail)

--- CARA 2: Via Command Line (headless) ---
  1. Buka terminal/command prompt
  2. Jalankan:

     godot --headless -s addons/gut/gut_cmdln.gd \
           --path D:\GBEU\g-beat-em-up \
           -gtest="res://test/" \
           -ginclude_subdirs \
           -gexit

     Penjelasan flag:
       --headless          : tanpa window
       -s addons/gut/...   : script entry GUT CLI
       --path <dir>        : path project
       -gtest="res://test/" : direktori test
       -ginclude_subdirs   : cari test di subfolder
       -gexit              : keluar setelah selesai

--- CARA 3: Run test spesifik ---
  godot --headless -s addons/gut/gut_cmdln.gd \
        --path D:\GBEU\g-beat-em-up \
        -gselect="test_player_inventory.gd" \
        -gexit

--- CARA 4: Export hasil ke JUnit XML ---
  godot --headless -s addons/gut/gut_cmdln.gd \
        --path D:\GBEU\g-beat-em-up \
        -gjunit_xml_file="res://test/test_results.xml" \
        -gexit

================================================================================
 MENAMBAH TEST BARU
================================================================================

  1. Buat file baru di res://test/ dengan prefix "test_"
  2. extends GutTest
  3. Tulis fungsi test diawali "test_"
  4. Jalankan ulang GUT

  Contoh struktur:
    extends GutTest

    func before_each():
        # setup sebelum tiap test

    func test_something():
        var result = my_function(arg)
        assert_eq(result, expected_value)

================================================================================
 CATATAN
================================================================================

- Autoloads (PlayerInventory, SurvivalStats, dll) bisa diakses langsung
  dari test tanpa perlu instance.
- Untuk class biasa (non-autoload), buat instance dengan ClassName.new()
- Resource class (SaveData, PlayerConfig, dll) bisa diinstantiate langsung.
- Test untuk scene/Node memerlukan autofree atau queue_free di after_each()
  agar tidak meninggalkan orphan nodes.
- File .tres testing dapat diletakkan di res://test/fixtures/ jika diperlukan.
